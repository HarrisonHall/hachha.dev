At its core, Radicle is a decentralized git forge. So GitHub, but without
Microsoft. Since it isn't centralized, the risk of having useful software taken
down by overeager legal teams should be far lower. Radicle still supports
private repos. It uses a custom P2P protocol based off of Secure Scuttlebutt
(SSB).

## Local Node: Setting up Radicle

A few ideas are necessary to understand at the get-go to reduce the cognitive
overhead of setting up your "account".

- _Radicle_ is a _decentralized git forge_.
  - Radicle supports PRs (==patch== requests), issues, and other collaboration
    which can be done with just the CLI, or with desktop/web interfaces.
  - Radicle nodes form a distributed network that ensures a repo is not on a
    single server.
  - Radicle is fairly viable but features that may be necessary for your
    adoption may not yet be implemented.
- Every device that connects to the Radicle network must have a running
  ==node==.
  - Nodes are essentially independent accounts and are **not _currently_
    shareable across devices**, even by the same user.
  - Nodes are identified by their ==did== (decentralized identifier).
- The Radicle network is decentralized but there are still repository "owners"
  (==delegates==).
  - Delegates must agree on changes according to the associated policy.
  - Since nodes are not (yet) cross-device, you'll need to be a delegate from
    each machine that you work from.

Radicle has several detailed official guides online, but I think it is useful to

daemon.

## Seed Nodes: Setting up Your Server

My home server is my seed node.

```
sudo groupadd --system seed
sudo useradd --system --gid seed --create-home seed
```
