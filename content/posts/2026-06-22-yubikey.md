I recently got a YubiKey after attending a
[talk at LinuxFest gnu/south](https://www.youtube.com/@southeastlinuxfest/videos).
The talk sold me on purchasing a YubiKey, but after receiving one it wasn't
clear to me _how_ to use it correctly. The key can do a lot, and while the
website "has" plenty of documentation, it essentially boils down to each
individual use-case for how to use it correctly.

I had a few needs I wanted solved:

1. Have a portable SSH key
2. Have a secure signing key

## System Preconditions

Getting this set up on my desktop and laptop was "unnecessarily fun".
Ultimately, what I think you essentially need for your Linux system is:

- GPG Agent
- `ykman`
- `YubiCo Authenticator`

### NixOS

Install these packages: `yubikey-manager` (`ykman`), `yubico-piv-tool` ,
`opensc`, `p11-kit`, `libfido2`.

```nix
# Enable gnupg agent (GPG Agent GNU Privacy Guard):
programs.gnupg.agent = {
  enable = true;
  enableSSHSupport = true;
};

# Smart card support:
services.pcscd.enable = true;
environment.etc."pkcs11/modules/opensc-pkcs11".text = ''
  module: ${pkgs.opensc}/lib/opensc-pkcs11.so
'';
programs.ssh.agentPKCS11Whitelist = "${pkgs.opensc}/lib/opensc-pkcs11.so";
programs.firefox.policies.SecurityDevices.p11-kit-proxy = "${pkgs.p11-kit}/lib/p11-kit-proxy.so";

# AFAIK the source is deprecated but the package is necessary on nix due to some
# udev rule shenanigans.
services.udev.packages = [ pkgs.yubikey-personalization ];
```

I installed `Yubico Authenticator` via Flatpak.

### Initializing the YubiKey

Before doing too much, you should update various PINs on the key. This can be
done via `ykman` and `Yubico Authenticator`.

## Uses & Setup

### SSH Key (PIV): Portable SSH Key

> [!WARNING]  
> There are considerations to generating the SSH key on the YubiKey itself.
> TODO: link

Use the Yubico Authenticator app to generate a self-signed certificate in
certificate slot 9a. Dump the certificate to a file `certificate-9a.crt`.

- Dump your public key: `ykman piv keys export -v 9a id_9a.pem`
- Regenerate your SSH public key:
  `ssh-keygen -i -m PKCS8 -f id_9a.pem > piv-9a.yubi.pub`
- Copy key into `~/.ssh`: `cp piv-9a.yubi.pub ~/.ssh/.`

Update `~/.ssh/config` to use key:

```ssh
Host my-server
  HostName <ip>
  User my-user
  Port 22
	IdentityFile ~/.ssh/piv-9a.yubi.pub
```

### GPG Signing Key: Sign Commits

Follow
[this guide](https://www.spletzer.com/2026/04/a-no-nonsense-guide-to-gpg-commit-signing-with-a-yubikey/).

Before you get started, there are a few notes:

- You should prepare to run a live image on a machine that can disconnect from
  the internet
- Ensure you have a means of saving the public key and any other material, this
  does not regenerate after you have your private key(s) on your YubiKey

### SSH Key (FIDO): Secure SSH Key

FIDO SSH keys are neat.

Pro: Create an ssh-key for a single machine that requires the YubiKey to be able
to use.

Con: Not portable, requires redoing process when migrating between machines.

`-sk` is for "security key."

- Create:
  `ssh-keygen -t ed25519-sk -O resident -O verify-required -O application=ssh:personalkey -C "ssh: hachha@hachha.dev"`
- Copy: `cp id_ed25519_sk ~/.ssh/mykey.yubi`
- SSH: `ssh -i ~/.ssh/mykey.yubi my-user@my-server -o IdentityAgent=none`

Update `~/.ssh/config` to use key:

```ssh
Host my-server
  HostName <ip>
  User my-user
  Port 22
  IdentityFile ~/.ssh/mykey.yubi
  IdentityAgent none
```

## Application Usage

### Utils

- `ykman` - CLI tool for managing key
- `Yubico Authenticator` - Graphical tool for managing key

### Jujutsu

Sign commits with `jj sign -r <rev>`.

```toml
[signing]
behavior = "drop"
backend = "gpg"
key = "<key-id>"
```

> [!TIP]  
> You can find your key id via the fingerprint in `ykman openpgp keys info sig`.
